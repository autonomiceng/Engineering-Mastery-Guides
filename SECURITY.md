# Security Policy

## Scope

This repository primarily contains documentation, generated text, and infrastructure examples.
Security reports are still useful, especially when they involve:

- Secrets or credentials committed by mistake
- Insecure infrastructure defaults in example code
- Dangerous automation that could mislead readers into unsafe deployments
- Supply-chain concerns in GitHub Actions or repository configuration

## Reporting

Please report security issues privately by opening a GitHub security advisory if the repository is hosted on GitHub with that feature enabled.
If that is not available, contact the maintainer directly before opening a public issue.

## What To Include

- A clear description of the issue
- Affected file paths
- Reproduction steps, if applicable
- Operational impact
- Suggested remediation, if known

## Expectations

- Do not publish exploitable details before the issue is reviewed.
- Do not assume example code is production-safe without manual validation.
- Treat all infrastructure snippets in this repository as review-required material.
